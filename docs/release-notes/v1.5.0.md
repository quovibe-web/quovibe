⚠️  **Breaking change — database layout**
v1.5.0 ships ADR-015: each portfolio now lives in its own SQLite file under `data/portfolios/<uuid>.db`, plus new `vf_*` tables (`vf_portfolio_meta`, `vf_dashboard`, `vf_chart_config`,
`vf_csv_import_config`, `vf_exchange_rate`). Pre-1.5 single-DB installs are **not compatible** — boot logs a one-time warning on a legacy `portfolio.db` but does not migrate it. To upgrade:
re-import your portfolio via PP-XML on the new Welcome page, or start fresh. Back up your old DB first.
`.db` files exported from v1.5 (Settings → Data → Export) are forward-compatible and re-importable on any v1.5+ install via the `/import` hub.

## Features

### Portfolio-per-DB (ADR-015)
- **Multi-portfolio support**: each portfolio in its own SQLite file, swappable at runtime.
- **Portfolio switcher** in TopBar (recency sort, "All portfolios" entry).
- All routes scoped under `/p/:portfolioId/*`; portfolio-scoped React Query keys.
- **Refcounted LRU pool** for DB handles; full cross-portfolio isolation (ADR-016).
- **SSE event bus** (`/api/events`) for cross-tab sync.
- `bootstrap.sql` as single DDL source of truth, parity-checked at CI.

### Welcome / Import / Setup
- New **Welcome page** with three bootstrap cards: fresh, demo, import.
- Dedicated **`/import` hub** for PP-XML and `.db` imports with rename-on-collision.
- **Demo portfolio** with mutex-guarded singleton; seeded with USD + FX flows.
- **Fresh portfolio setup** (M3 dialog, multi-deposit + securities-account seeding); legacy N=0 redirect.
- **Portfolio settings page**: rename / delete / export / data tab.
- Duplicate-name guard across all create paths.

### Charts
- Multi-pane **PerformanceChart** with interactive legend, color/style picker, zero-crossing gradient fill.
- **PriceChart** with OHLC, native transaction markers (distinct shapes for buy/sell/dividend), reporting-period filter.
- Migrated `WidgetPerfChart`, `WidgetDrawdownChart`, `TaxonomySeries`, `Payments` to lightweight-charts.
- Zero-dependency canvas `Sparkline` for hero + Movers.
- Chart toolbar portal in card header; theme-aware via `useChartColors`.

### Dashboard
- **Four-zone layout**: hero, metrics strip, main widgets, detail zone.
- `DashboardHero` with live sparkline; configurable `DashboardMetricsStrip`.
- **Multi-dashboard / tabbed dashboards** per portfolio.
- Configurable grid columns; widget catalog with per-widget hide/show.
- Widget migration registry; unsupported-widget placeholder.
- Empty state with template cards; staggered zone reveal.

### UI / UX
- New warm palette (parchment light / warm near-black dark); 8-color chart palette tuned for both.
- **NumberFlow** animated digits across `CurrencyDisplay`, `MetricCard`, percentage widgets.
- **Frosted-glass sticky TopBar** on scroll.
- **Framer Motion** sliding `SegmentedControl` and tab indicators.
- Card hover lift / border accent; rounded-full pills on toggles.
- Sidebar polish; mobile-responsive TopBar with drawer for theme/lang.
- New aperture-mark logo + serif/sans wordmark.
- Multi-currency cash breakdown on summary cards.
- Broker-style open-position PnL on brokerage cards.

### Watchlists
- 1D / 1W / 1M / 1Y price-change columns; manual instrument creation.

### Logo resolver (ADR-013)
- `POST /api/logo/resolve` stateless endpoint.
- ETF / fund branching with 37+ providers (Vanguard, iShares, ARK, Schwab, Global X, ProShares, Direxion, LGIM, HANetf, 21Shares, DFA…).
- CoinGecko / Yahoo / favicon strategy chain.
- Auto-fetch on security + account creation.

### CSV import — full PP-CSV parity
- **Cross-currency**: `Exchange Rate`, `Gross Amount`, `Currency Gross Amount`; FX-aware `xact_unit` for BUY / SELL / DIVIDEND / TRANSFER; PP step-2 verification.
- **Foreign-currency fees / taxes** → FOREX FEE/TAX units.
- **Per-row account routing** (4 columns) — single CSV across multiple brokers.
- **Default-Type inference** when `type` column unmapped.
- **Re-import dedupe** — partial unique index on natural key, idempotent.
- **Volume suffix** (`K`/`M`/`B`) on price flow only.
- **New-security currency** from `Currency Gross Amount` (no hardcoded EUR).
- Accept-and-ignore for `WKN`, `Time`, `Date of Quote`.
- Inner-securities-account picker step.

### Attribute Types
- Full CRUD with PP parity — only `logo` builtin-protected; all PP-seeded ids editable.
- RHF + zodResolver dialog; cascade delete.

### FX
- Centralized **fx-scheduler** with PP-aligned cadence, eager-fire on first portfolio open.
- Lazy fetch on cache miss for cross-currency BUY/SELL prefill.

### Forms
- **`useGuardedSubmit`** — synchronous re-entry guard, wired across 13 dialogs.
- **TransactionForm migrated to RHF + zodResolver** (per-field validation, Save gated on validity).
- All Edit*Dialog variants unified onto `TransactionForm` wrappers.

### i18n / accessibility
- 8 languages (en, it, de, fr, es, nl, pl, pt) covered for every new namespace.
- Server error codes translated in UI via centralized `errors.json`.
- `NumberFlow` + `CurrencyDisplay` expose values to screen readers.
- lightweight-charts locale bound to `i18n.language`.

### Operations
- Image healthcheck, compose limits, CI bootstrap parity gate.
- Per-portfolio backup / restore via `db.backup()`.
- Boot-recovery for missing files / orphan registry / demo re-materialization.

### Production-readiness (v1.5.0 audit pass)
- Docker pinned to Alpine 3.20 + pnpm 9.0.0 + py3-lxml; build-time smoke tests.
- `engines` field in root `package.json` (node ≥24, pnpm ≥9).
- Per-DB `schemaVersion` read-check at open; mismatches → 503 with discriminable code.
- Orphan `.db` handling: `deletePortfolio` returns warnings; boot-time `sweepOrphanPortfolios` + `sweepOrphanWalShm`.
- Cross-process `instance-lock` on `DATA_DIR` — refuses dual-process startup; stale-lock reaping.
- `SQLITE_FULL` / `SQLITE_IOERR_WRITE` → HTTP 507 `INSUFFICIENT_STORAGE`.
- CSV / XML / export temp paths moved from `os.tmpdir()` to `DATA_DIR/tmp` — container-restart safe.
- Graceful SIGTERM drain (`server.close` awaited, configurable timeout).
- Web bundle: `console` + `debugger` dropped in production builds; no source maps.
- CI tag workflow re-runs the full gate suite (build + lint:engine + governance + arch + bootstrap + tests) against the tagged tree before pushing the image.
