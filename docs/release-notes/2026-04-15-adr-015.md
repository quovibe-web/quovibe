# quovibe — ADR-015 release

## Breaking changes

- **Pre-ADR-015 installs are not migrated.** Any pre-existing `data/portfolio.db` is ignored at boot. To keep your data, export to PP XML before upgrading and re-import via the new Welcome screen.
- `data/schema.db` is removed from the repo and no longer copied at first boot.
- The `PriceScheduler` worker and its cron schedule (`PRICE_CRON_SCHEDULE` env) are removed. Prices are fetched on demand (button) or optionally on first pool-open per portfolio (Settings → Data, default OFF).
- The `SCHEMA_PATH` env is removed.
- Frontend routes: every portfolio-aware page now lives under `/p/:portfolioId/*`. Legacy paths redirect via the router where they were well-defined.

## New features

- Multi-portfolio architecture with URL-scoped routing (multi-tab safe).
- Welcome screen with three bootstrap options (Import from PP / Try Demo / Start Fresh).
- Portfolio switcher in the top bar.
- Portfolio export and import as raw `.db` files.
- Per-portfolio dashboards and chart configuration travel with the portfolio file.
- Server-Sent Events channel (`/api/events`) for cross-tab portfolio lifecycle notifications.

## Operator notes

- Docker: mount `/app/data` as a volume. The demo source now lives at `/app/assets/demo.db` (outside the volume) so a fresh volume mount doesn't hide it.
- Reverse proxies: `GET /api/events` is an SSE stream; configure your proxy to NOT buffer that path. Example for nginx:
  ```
  location /api/events {
    proxy_buffering off;
    proxy_cache off;
    proxy_read_timeout 24h;
  }
  ```
- Settings sidecar (`data/quovibe.settings.json`) now carries `schemaVersion: 1`. Downgrading the app past ADR-015 will refuse to start; restore a prior sidecar from backup.
