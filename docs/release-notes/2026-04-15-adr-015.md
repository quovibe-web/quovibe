# quovibe — ADR-015 release

## Upgrade path (read before upgrading)

**Auto-migration does not exist.** If you have a pre-ADR-015 install:

1. On the old version, export your portfolio to **Portfolio Performance XML** via Settings → Data.
2. Upgrade.
3. On the new Welcome screen, pick **Import from Portfolio Performance** and re-upload the XML.

A stale `data/portfolio.db` sitting next to the upgraded app is silently ignored — the new boot-recovery only looks for `portfolio-{uuid}.db` and `portfolio-demo.db` filenames (see ADR-015 §3.5).

## Breaking changes

- **Pre-ADR-015 installs are not migrated.** Any pre-existing `data/portfolio.db` is ignored at boot. See **Upgrade path** above.
- **Sidecar `dashboards` and `activeDashboard` fields are dropped silently on first load.** Dashboards now live per-portfolio in each `.db`'s `vf_dashboard` table. Old sidecar dashboard definitions cannot be auto-migrated (no portfolio scope). Export to PP XML before upgrading to preserve data; then rebuild dashboards in the new UI.
- **`csv_import_config` table renamed to `vf_csv_import_config`.** External tools that read the quovibe SQLite directly must update table names. The in-app CSV import flow is unaffected.
- `data/schema.db` is removed from the repo and no longer copied at first boot.
- The `PriceScheduler` worker and its cron schedule (`PRICE_CRON_SCHEDULE` env) are removed. Prices are fetched on demand (button) or optionally on first pool-open per portfolio (Settings → Data, default OFF).
- The `SCHEMA_PATH` env is removed. The `DB_PATH` module-level constant is removed.
- Frontend routes: every portfolio-aware page now lives under `/p/:portfolioId/*`. Legacy paths redirect via the router where they were well-defined.

## New features

- Multi-portfolio architecture with URL-scoped routing (multi-tab safe).
- Welcome screen with three bootstrap options (Import from PP / Try Demo / Start Fresh).
- Portfolio switcher in the top bar.
- Portfolio export and import as raw `.db` files.
- Per-portfolio dashboards and chart configuration travel with the portfolio file.
- Server-Sent Events channel (`/api/events`) for cross-tab portfolio lifecycle notifications.

## Operator notes

- Docker: mount `/app/data` as a volume. The demo source now lives at `/app/assets/demo.db` (outside the volume) so a fresh volume mount doesn't hide it. Override the source path via the `QUOVIBE_DEMO_SOURCE` env var (defaults to `/app/assets/demo.db` in Docker, `<repo>/data/demo.db` in dev).
- Reverse proxies: `GET /api/events` is an SSE stream; configure your proxy to NOT buffer that path. Example for nginx:
  ```
  location /api/events {
    proxy_buffering off;
    proxy_cache off;
    proxy_read_timeout 24h;
  }
  ```
- Settings sidecar (`data/quovibe.settings.json`) now carries `schemaVersion: 1`. Downgrading the app past ADR-015 will refuse to start; restore a prior sidecar from backup.
