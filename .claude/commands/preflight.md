# Preflight Check — Governance 2.0

Run `pnpm preflight` and report the output to the user.

## Interpretation

**❌ (blocking errors):** list problems, propose fixes, do NOT proceed.

**Only ✅ and/or ⚠️:** report relevant warnings, then:
- Read `docs/architecture/README.md` to identify relevant architecture files
- Read `docs/CHANGELOG-SESSIONS.md` for session context

## Confirm objective

Summarize in 2-3 sentences: what you will do, which packages you will touch, which tests you will add.
**Wait for user confirmation before proceeding.**
